孵化器核心价值

适应社区治理模式

调用社区核心资源

培育为正式项目

通过导师指导、社区反馈优化，帮助项目达到 Dromara 正式项目标准，完成"孵化-毕业"全流程。

孵化器指南

前置条件寻找导师编写提案提交评审孵化毕业

1. 前置条件评估

确认项目符合孵化基本要求，这是申请的基础条件：

项目为原创，非 Fork 版本，拥有独立的知识产权

提供完整的应用解决方案（非单一工具类组件），具备实际应用价值

代码注释规范清晰，使用文档持续完善，方便新用户快速上手

近 2 个月内有实际有效的代码提交记录，保持项目活跃度

若项目不符合以上条件，建议先完善后再提交申请，以提高通过率。

2. 寻找项目导师

匹配 1-3 位社区导师，获取专业指导：

从社区导师列表中选择适配项目领域的导师，获取孵化流程指导。每个项目至少需 1 位导师，推荐 3 位以覆盖多维度支持。

导师职责：
  • 指导项目熟悉社区治理模式和流程
  • 协助完善项目文档和代码规范
  • 提供技术方向建议和社区资源对接
  • 协助准备和优化项目提案

3. 编写项目提案

与导师协作完成提案，确保内容完整，以通过 Dromara TOC 委员会评审：

提案要素说明
项目名称清晰、无歧义的项目全称
项目描述项目定位、核心功能、解决的问题
社区参与度现有贡献者数量、用户反馈、issue 处理情况
代码地址GitHub/Gitee 等代码仓库链接（需公开可访问）
核心人员项目负责人、核心贡献者信息（联系方式可选）
License开源协议（需符合 Dromara 社区协议规范，如 Apache-2.0）

4. 提交提案评审

在 GitHub Issue 提交提案，等待 TOC 委员会评审：

操作步骤
1

访问 Dromara 孵化器 GitHub 仓库

2

新建 Issue，标题格式：【Proposal】xxx项目申请加入Dromara 开源组织孵化器

3

正文粘贴完整提案内容，提交后等待 TOC 委员会评审

前往提交提案

5. 成为孵化项目

评审通过后正式进入孵化阶段，获得社区资源支持：

TOC 委员会审核通过后，项目将被纳入 Dromara 孵化项目列表，获得以下支持：

基础设施支持

提供 CI/CD 服务、代码质量检测、自动部署等工具支持

社区推广

在社区渠道进行项目展示，帮助吸引更多用户和贡献者

交流渠道

加入社区核心开发者交流群，获取第一手社区动态

毕业指导

制定项目成长计划，指导项目达到正式项目标准

孵化期通常为 3-6 个月，期间导师将持续提供指导，帮助项目成长。
